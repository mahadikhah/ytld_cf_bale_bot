## 📌 معرفی پروژه

این پروژه یک ربات چندمنظوره برای پیام‌رسان **Bale** (و پل تلگرام) است که می‌تواند علاوه بر دانلود ویدیوهای یوتیوب، کارهای زیر را نیز انجام دهد:

- 🎵 **جستجو و دانلود موسیقی** (تکی و دسته‌ای) از یوتیوب
- 📄 **جستجوی مقالات علمی** در arXiv و دانلود PDF
- 🌦️ **پیش‌بینی آب‌وهوا** (ساعتی و هفتگی)
- 📦 **ذخیره‌سازی صفحات وب** به صورت فایل HTML کامل
- 📚 **آینه‌سازی مستندات** (دانلود تمام صفحات یک بخش)
- 🐙 **کاوش گیت‌هاب** (جستجوی مخزن‌ها، مرور فایل‌ها، ایشوها و PRها)
- 🌐 **جستجوی وب**
- 🔄 **پل تلگرام** – انتقال فایل از تلگرام به Bale (حتی فایل‌های بزرگ با اکانت کاربری)
- ☁️ **ارسال به S3 Object storatge** (اختیاری، برای زمان‌هایی که Bale قطع است)
- 💎 **اشتراک ویژه** (اولویت در صف دانلود)

برای دور زدن محدودیت‌های منابع رایگان، از دو سرویس رایگان استفاده شده است:

- **Cloudflare Workers**: برای دریافت پیام‌ها (Bale با polling و تلگرام با webhook)، مدیریت صف، و اجرای دستورات سبک.
- **GitHub Actions**: برای عملیات سنگین (دانلود ویدیو/موسیقی، دریافت صفحات، تبدیل فرمت، ارسال فایل‌های حجیم).

## 🛠️ پیش‌نیازها

1. **یک حساب کاربری در [GitHub](https://github.com)**.
2. **یک حساب کاربری در [Cloudflare](https://cloudflare.com)**.
3. **یک ربات Bale** – توکن از [BotFather](https://bale.ai/botfather).
4. **یک ربات تلگرام (اختیاری)** – توکن از [@BotFather](https://t.me/BotFather) (برای پل تلگرام).
5. **یک فایل کوکی معتبر یوتیوب** – فایل `cookies.txt` با فرمت Netscape.
6. **یک Personal Access Token (Classic)** گیت‌هاب با دسترسی `workflow` و `repo`.
7. **شناسه عددی حساب Bale** (Chat ID) – از ربات `@userinfobot`.
8. (برای انتقال فایل‌های بزرگ تلگرام) یک شماره تلفن (قابل حذف بعد از تولید نشست).

## 📁 فایل‌های پروژه

- **`bot_action.py`** – اسکریپت اصلی پایتون برای دانلود یوتیوب و موسیقی.
- **`tg_download.py`** – دانلود فایل‌های بزرگ تلگرام (با اکانت کاربری).
- **`bot_utils.py`** – توابع کمکی (نوار پیشرفت، پیام‌رسان Bale/Telegram).
- **پوشه `worker/`** – کد Cloudflare Worker (مدیریت پیام‌ها، فراخوانی workflowها).
- **پوشه `.github/workflows/`** – فایل‌های گردش‌کار گیت‌هاب:
  - `bot.yml` – دانلود یوتیوب، موسیقی
  - `getpage.yml` – ذخیره صفحه وب
  - `getdocs.yml` – آینه‌سازی مستندات
  - `paper_download.yml` – دانلود مقاله
  - `telegram_transfer.yml` / `telegram_transfer_large.yml` – انتقال فایل تلگرام
  - `gh_download.yml` – دانلود مخزن گیت‌هاب
  - `s3_cleanup.yml` – پاکسازی خودکار فایل‌های S3
  - `deploy-worker.yml` – دیپلوی خودکار Worker

## 🚀 گام‌های راه‌اندازی

### گام ۱: Fork کردن مخزن

به `https://github.com/mahadikhah/ytld_cf_bale_bot` بروید و **Fork** کنید.

### گام ۲: تنظیم Secrets در گیت‌هاب

**Settings → Secrets and variables → Actions** و اضافه کردن موارد زیر:

| نام Secret               | توضیحات |
|--------------------------|---------|
| `BALE_BOT_TOKEN`         | توکن ربات Bale |
| `ACTION_GITHUB_TOKEN`    | Personal Access Token گیت‌هاب (Classic) |
| `ACTION_GITHUB_REPO`     | نام کامل مخزن (مثلاً `yourname/ytld_cf_bale_bot`) |
| `BALE_PAYMENT_TOKEN`     | توکن پرداخت Bale (در صورت نیاز) |
| `ADMIN_CHAT_ID`          | شناسه عددی مدیر |
| `WORKER_SECRET`          | رمز تصادفی برای ارتباط Worker و Actions |
| `WORKER_URL`             | آدرس Worker (بعد از دیپلوی) |
| `YOUTUBE_COOKIES_B64`    | محتوای `cookies.txt` کد شده با Base64 |
| `CLOUDFLARE_API_TOKEN`   | API Token با دسترسی Workers و KV |
| `CLOUDFLARE_ACCOUNT_ID`  | Account ID کلاودفلر |
| `BOT_STATE_KV_ID`        | شناسه KV namespace وضعیت |
| `USER_PLANS_KV_ID`       | شناسه KV namespace کاربران |
| `TELEGRAM_BOT_TOKEN`     | (اختیاری) توکن ربات تلگرام |
| `ENABLE_S3`              | (اختیاری) `true` برای فعال‌سازی آپلود S3 |
| `S3_ACCOUNT_1_...` تا `5`| اطلاعات حساب‌های S3 (در صورت فعال بودن) |
| `TG_USER_SESSION`        | (اختیاری) نشست اکانت تلگرام (برای فایل‌های بزرگ) |

### گام ۳: ایجاد KV Namespace

در Cloudflare دو Namespace بسازید (`BOT_STATE` و `USER_PLANS`) و شناسه‌ها را در secrets قرار دهید.

### گام ۴: دیپلوی Worker

از تب Actions، workflow **Deploy Cloudflare Worker** را اجرا کنید. آدرس Worker را در secret `WORKER_URL` ذخیره کنید.  
**وب‌هوک تلگرام** به طور خودکار در همین workflow تنظیم می‌شود.

### گام ۵: فعال‌سازی پل تلگرام (اختیاری)

1. به ربات Bale دستور `/link telegram` بدهید. کدی دریافت می‌کنید.
2. در ربات تلگرام، دستور `/start <کد>` را بفرستید.
3. اکانت‌ها متصل می‌شوند و می‌توانید فایل‌ها را فوروارد کنید.

### گام ۶: تست ربات

- `/start` – پیام خوش‌آمد.
- یک لینک یوتیوب بفرستید → کیفیت‌ها نمایش داده می‌شوند.
- `/music <نام آهنگ>` – جستجوی موسیقی.
- `/batchmusic` – دانلود دسته‌ای (هر خط یک آهنگ).
- `/paper <عنوان>` – جستجوی مقاله.
- `/weather <شهر>` – پیش‌بینی آب‌وهوا.
- `/getpage <آدرس>` – ذخیره صفحه وب.
- `/getdocs <آدرس>` – آینه‌سازی مستندات.
- `/gh search <کوئری>` – جستجوی مخزن گیت‌هاب.
- `/link telegram` – اتصال به تلگرام.

## 📖 راهنمای استفاده

### یوتیوب
1. لینک را بفرستید → کیفیت‌ها ظاهر می‌شوند.
2. کیفیت را انتخاب کنید → در صورت فعال بودن S3، انتخاب روش ارسال (Bale / S3).
3. فایل‌های zip را دریافت کنید. همه را در یک پوشه گذاشته و فقط فایل `.zip` را باز کنید.

### موسیقی
- `/music <نام>` – ۵ نتیجه نمایش داده می‌شود، یکی را انتخاب کنید.
- `/batchmusic` سپس هر خط یک عنوان ارسال کنید (حداکثر ۱۰ آهنگ).

### مقالات علمی
- `/paper <عنوان>` – ۱۰ نتیجه با دکمه دانلود PDF. دکمه‌های صفحه‌بندی.

### آب‌وهوا
- `/weather <شهر>` – ۲۴ ساعت آینده و پیش‌بینی ۷ روزه. دکمه‌های «قبلی/بعدی».

### صفحات وب
- `/getpage <URL>` – فایل HTML کامل با تمام assets به صورت zip دریافت می‌کنید.

### مستندات
- `/getdocs <URL>` – تمام صفحات زیرمسیر به صورت یک فایل zip آینه می‌شود. زمان اجرا قابل تنظیم است.

### گیت‌هاب
- `/gh search <کوئری>` – جستجوی مخزن.
- `/gh repo owner/repo` – جزئیات، browse فایل‌ها، ایشوها، PRها، و دانلود.

### پل تلگرام
1. با `/link telegram` در Bale کد بگیرید.
2. در تلگرام `/start <کد>` بفرستید.
3. حالا هر پیام (متن، عکس، ویدیو، فایل) را در تلگرام فوروارد کنید → در Bale دریافت می‌شود.
4. فایل‌های بزرگ (>20MB) از طریق یک کانال خصوصی و اکانت کمکی منتقل می‌شوند.

## 🧩 رفع مشکلات متداول

- **ربات پیام نمی‌خواند** → Worker دیپلوی شده و Cron فعال است؟ لاگ‌های Cloudflare را بررسی کنید.
- **خطای دانلود یوتیوب** → کوکی‌ها معتبر نیستند، دوباره استخراج کنید.
- **فایل zip ارسال نمی‌شود** → مطمئن شوید فایل نهایی از ۲۰MB کمتر است (در غیر این صورت قطعه‌قطعه می‌شود).
- **صف دانلود قفل می‌ماند** → Worker همیشه در پایان `/github/done` را صدا می‌زند. در صورت قفل ماندن، می‌توانید دستی با POST به `/github/done` همراه `secret` و `chat_id` باز کنید.
- **انتقال فایل بزرگ تلگرام خطا می‌دهد** → نشست اکانت کمکی منقضی شده، دوباره با Colab تولید کنید (و حتماً Runtime را قطع کنید).
- **سرعت دانلود تلگرام کم است** → محدودیت فیزیکی شبکه بین GitHub Actions و سرورهای تلگرام. با ۸ thread موازی به ~5 MB/s می‌رسید.

## 💡 نکات تکمیلی

- **امنیت**: `WORKER_SECRET` را قوی انتخاب کنید.
- **هزینه**: Cloudflare Workers رایگان (100K درخواست/روز)، GitHub Actions برای مخازن عمومی رایگان.
- **شخصی‌سازی**: می‌توانید `MAX_FILE_SIZE`، تعداد thread های موازی، یا تعداد نتایج جستجو را تغییر دهید.
- **S3**: با فعال‌سازی `ENABLE_S3`، فایل‌ها به ArvanCloud یا هر S3 سازگار آپلود می‌شوند و لینک موقت دریافت می‌کنید.
