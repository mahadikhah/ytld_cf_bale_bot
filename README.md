
## 📌 معرفی پروژه

این پروژه یک ربات تلگرامی (مخصوص پیام‌رسان **Bale**) است که می‌تواند لینک‌های یوتیوب را دریافت کرده، کیفیت‌های مختلف ویدیو را نمایش دهد و در نهایت ویدیوی انتخاب‌شده را دانلود و به صورت فایل‌های زیپ چندتکه برای کاربر ارسال کند.

برای دور زدن محدودیت‌های منابع رایگان، از دو سرویس رایگان استفاده شده است:

- **Cloudflare Workers**: برای دریافت پیام‌های کاربران از Bale (با polling) و مدیریت صف دانلود.
- **GitHub Actions**: برای اجرای عملیات سنگین دانلود با `yt-dlp` (چون Cloudflare Worker محدودیت زمان اجرا دارد).

شما با دنبال کردن مراحل زیر می‌توانید این ربات را برای خودتان راه‌اندازی کنید.

---

## 🛠️ پیش‌نیازها

1. **یک حساب کاربری در [GitHub](https://github.com)**.
2. **یک حساب کاربری در [Cloudflare](https://cloudflare.com)**.
3. **یک ربات Bale** که توکن آن را از [BotFather](https://bale.ai/botfather) دریافت کرده‌اید.
4. **یک فایل کوکی (cookies.txt) معتبر برای یوتیوب** (اختیاری ولی برای برخی ویدیوها ضروری است).  
   - با افزونه‌هایی مثل "Get cookies.txt LOCALLY" می‌توانید کوکی‌های مرورگر خود را استخراج کنید.

🔗 **مخزن رسمی گیت‌هاب:**  
[https://github.com/kairi003/Get-cookies.txt-LOCALLY](https://github.com/kairi003/Get-cookies.txt-LOCALLY)

**لینک نصب مستقیم از فروشگاه‌های رسمی:**

- **کروم / اج / بریو:**  
  [Chrome Web Store - Get cookies.txt LOCALLY](https://chromewebstore.google.com/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc)

- **فایرفاکس:**  
  [Firefox Add-ons - Get cookies.txt LOCALLY](https://addons.mozilla.org/en-US/firefox/addon/get-cookies-txt-locally/)

این افزونه کوکی‌ها را با فرمت استاندارد Netscape (مورد نیاز yt-dlp) ذخیره می‌کند و برخلاف افزونه‌های مشابه، کوکی‌ها را به سرور خارجی ارسال نمی‌کند (کاملاً محلی کار می‌کند).

5. **یک کلاسیک GitHub Personal Access Token (Classic)** با دسترسی `workflow` و `repo`.

6. **شناسه عددی حساب Bale خود** (Chat ID) – می‌توانید از ربات `@userinfobot` در Bale استفاده کنید.

## 📁 فایل‌های پروژه

پروژه شامل سه بخش اصلی است:

- `bot_action.py`: اسکریپت پایتون که در GitHub Actions اجرا می‌شود و کار دانلود را انجام می‌دهد.
- پوشه `worker`: کد Cloudflare Worker که ارتباط با Bale و مدیریت صف را بر عهده دارد.
- فایل `bot.yml` در مسیر `.github/workflows`: گردش‌کار (workflow) گیت‌هاب که با فراخوانی Worker اجرا می‌شود.

## 🚀 گام‌های راه‌اندازی

### گام ۱: Fork کردن مخزن (Repository)

ابتدا به آدرس مخزن اصلی بروید (مثلاً `https://github.com/mahadikhah/ytld_cf_bale_bot`) و دکمه **Fork** را بزنید تا یک کپی از پروژه در اکانت گیت‌هاب خودتان ایجاد شود.

### گام ۲: تنظیم Secrets در گیت‌هاب

وارد مخزن Fork شده خود شوید و به مسیر **Settings → Secrets and variables → Actions** بروید. دکمه **New repository secret** را زده و مقادیر زیر را اضافه کنید:

| نام Secret             | توضیحات                                                                 |
|------------------------|--------------------------------------------------------------------------|
| `BALE_BOT_TOKEN`       | توکن ربات Bale که از BotFather گرفته‌اید.                                |
| `ACTION_GITHUB_TOKEN`  | Personal Access Token گیت‌هاب (Classic) با دسترسی‌های workflow و repo.    |
| `ACTION_GITHUB_REPO`   | نام کامل مخزن شما (مثلاً `yourusername/ytld_cf_bale_bot`).               |
| `BALE_PAYMENT_TOKEN`   | توکن پرداخت Bale (اگر از قابلیت فروش اشتراک استفاده می‌کنید، در غیر این صورت می‌توانید خالی بگذارید). |
| `ADMIN_CHAT_ID`        | شناسه عددی اکانت Bale خودتان (مدیر ربات).                                |
| `WORKER_SECRET`        | یک رمز دلخواه برای ارتباط امن بین Worker و GitHub Actions (مثلاً یک رشته تصادفی). |
| `WORKER_URL`           | آدرس Worker شما در Cloudflare (مثلاً `https://bale-yt-bot.workers.dev`).  |
| `YOUTUBE_COOKIES_B64`  | محتوای فایل `cookies.txt` که با Base64 کد شده است (دستور زیر را در ترمینال اجرا کنید: `base64 -w 0 cookies.txt`). |
| `CLOUDFLARE_API_TOKEN` | API Token از Cloudflare با دسترسی ویرایش Workers و KV.                    |
| `CLOUDFLARE_ACCOUNT_ID`| Account ID حساب Cloudflare شما (در داشبورد Cloudflare پیدا می‌شود).       |
| `BOT_STATE_KV_ID`      | شناسه KV Namespace برای وضعیت ربات (در گام بعد ایجاد می‌شود).             |
| `USER_PLANS_KV_ID`     | شناسه KV Namespace برای اطلاعات کاربران (در گام بعد ایجاد می‌شود).        |

> **نکته مهم**: مقدار `WORKER_URL` را فعلاً می‌توانید یک آدرس موقت بگذارید، پس از دیپلوی Worker مقدار دقیق آن را جایگزین کنید.

### گام ۳: ایجاد KV Namespace در Cloudflare

1. وارد داشبورد Cloudflare شوید و از منوی سمت چپ **Workers & Pages** را انتخاب کنید.
2. به تب **KV** بروید و روی **Create namespace** کلیک کنید.
3. دو namespace با نام‌های دلخواه (مثلاً `BOT_STATE` و `USER_PLANS`) ایجاد کنید.
4. شناسه (ID) هر namespace را کپی کرده و در گیت‌هاب Secrets به ترتیب با نام‌های `BOT_STATE_KV_ID` و `USER_PLANS_KV_ID` ذخیره کنید.

### گام ۴: دیپلوی Worker روی Cloudflare

این کار به صورت خودکار توسط GitHub Actions انجام می‌شود، اما ابتدا باید اجازه دهید workflow مربوطه اجرا شود.

1. در مخزن Fork شده، به تب **Actions** بروید و اگر workflow ها غیرفعال هستند، آن‌ها را فعال کنید.
2. workflow با نام **Deploy Cloudflare Worker** را پیدا کرده و **Run workflow** را بزنید (شاخه `main` را انتخاب کنید).
3. پس از اتمام موفق، Worker شما در آدرسی مانند `https://bale-yt-bot.نام-زیردامنه-شما.workers.dev` در دسترس خواهد بود.
4. این آدرس را در Secret گیت‌هاب با نام `WORKER_URL` قرار دهید.

### گام ۵: فعال‌سازی Cron Trigger در Worker

Worker شما هر دقیقه یک بار اجرا می‌شود تا پیام‌های جدید را از Bale دریافت کند. این کار با تنظیم `crons = ["* * * * *"]` در فایل `wrangler.toml` انجام شده است و نیازی به کار اضافی نیست.

### گام ۶: تست ربات

حالا ربات شما آماده است! به Bale بروید و به ربات خود پیام `/start` بدهید. سپس یک لینک یوتیوب (مثلاً `https://youtu.be/xxxxxxx`) ارسال کنید. ربات باید کیفیت‌های موجود را به صورت دکمه‌های inline نمایش دهد. با انتخاب یک کیفیت، دانلود آغاز می‌شود و پس از چند دقیقه فایل‌های zip در چت ارسال می‌شوند.

## 📖 راهنمای استفاده برای کاربران نهایی

1. **شروع کار**: به ربات پیام `/start` بدهید.
2. **بررسی وضعیت اشتراک**: با دکمه "Check Premium Status" یا دستور `/status` می‌توانید وضعیت حساب خود را ببینید.
3. **خرید اشتراک (در صورت فعال بودن)**: با دستور `/buy` می‌توانید اشتراک premium تهیه کنید.
4. **ارسال لینک یوتیوب**: کافیست لینک ویدیو را در چت بفرستید.
5. **انتخاب کیفیت**: از بین گزینه‌های نمایش داده شده، کیفیت مورد نظر را انتخاب کنید.
6. **دریافت ویدیو**: پس از اتمام دانلود، چند فایل zip دریافت می‌کنید (مثلاً `VideoName_300.z01`, `VideoName_300.z02`, `VideoName_300.zip`).  
   - **نحوه باز کردن**: همه فایل‌ها را در یک پوشه قرار دهید و فقط فایل با پسوند `.zip` را باز کنید. سیستم به طور خودکار قطعات را ترکیب کرده و فایل MP4 نهایی را استخراج می‌کند.

## 🧩 رفع مشکلات متداول

### ۱. ربات پیام‌ها را نمی‌خواند
- مطمئن شوید Worker دیپلوی شده و Cron فعال است.
- بررسی کنید `BALE_BOT_TOKEN` صحیح باشد.
- از تب Logs در Cloudflare Dashboard (Workers & Pages → bale-yt-bot → Logs) خطاها را مشاهده کنید.

### ۲. خطای "Download failed" یا "Did not get any data blocks"
- فایل کوکی (`YOUTUBE_COOKIES_B64`) معتبر نیست یا منقضی شده. یک فایل جدید استخراج و Base64 آن را جایگزین کنید.
- ممکن است IP رانر گیت‌هاب توسط یوتیوب محدود شده باشد. استفاده از کوکی معتبر معمولاً مشکل را حل می‌کند.

### ۳. دانلود انجام می‌شود ولی فایل zip ارسال نمی‌شود
- حجم فایل نهایی بیشتر از ۲۰ مگابایت (حداکثر Bale) بوده و باید قطعه‌قطعه شود. اگر `zip` روی رانر گیت‌هاب نصب نباشد، خطا رخ می‌دهد. در workflow ما `zip` به صورت پیش‌فرض روی ubuntu-latest وجود دارد.

### ۴. صف دانلود قفل می‌ماند (کاربر نمی‌تواند دانلود جدید شروع کند)
- Worker پس از اتمام کار (چه موفق چه ناموفق) باید قفل را با فراخوانی endpoint `/github/done` باز کند. در کد جدید `bot_action.py` این درخواست همیشه در بخش `finally` ارسال می‌شود. اگر همچنان قفل ماند، می‌توانید به صورت دستی از طریق Worker URL با ارسال POST به `/github/done` با `secret` و `chat_id` آن را باز کنید.

## 💡 نکات تکمیلی

- **امنیت**: حتماً `WORKER_SECRET` را یک رشته تصادفی و قوی انتخاب کنید. این رمز باید در Secrets گیت‌هاب و در متغیر محیطی Worker یکسان باشد.
- **مدیریت هزینه**: Cloudflare Workers تا ۱۰۰,۰۰۰ درخواست در روز رایگان است. GitHub Actions نیز برای مخازن عمومی رایگان است.
- **شخصی‌سازی**: می‌توانید محدودیت حجم فایل (`MAX_FILE_SIZE`) را در `bot_action.py` تغییر دهید (حداکثر مجاز Bale ۲۰ مگابایت است).
